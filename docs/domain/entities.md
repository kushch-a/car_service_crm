# Аналіз домену та сутності

Система розділена на 4 основні Bounded Contexts (Обмежені контексти):

## 1. CRM Context (Управління клієнтами)
Відповідає за базу даних клієнтів та їхніх транспортних засобів.
* **Customer (Клієнт):** Власник авто. Ключова сутність.
* **Car (Автомобіль):** Об'єкт обслуговування. Має VIN-код, марку, модель.

## 2. Service Operations (Виконання робіт)
Основний операційний контекст. Планування та фіксація ремонтів.
* **ServiceRecord (Запис на сервіс):** Факт виконання роботи. Зв'язує Авто, Послугу та Майстра.
* **ServiceCatalogItem (Послуга):** Елемент прайс-листа (назва, ціна, нормо-години).
* **Mechanic (Майстер):** Роль користувача, що виконує роботу.

## 3. Billing (Фінанси)
Розрахунок вартості та виставлення рахунків.
* **Invoice (Рахунок):** Фінансовий документ для оплати.
* **InvoiceLineItem (Позиція рахунку):** Конкретна послуга в чеку з ціною на момент створення.

## 4. IAM (Identity & Access Management)
Автентифікація та права доступу.
* **User:** Користувач системи (Адмін, Менеджер, Механік).
* **Role:** Набір прав доступу.
