# 1. Вибір архітектурного стилю

* Статус: Прийнято
* Дата: 2025-11-28

## Контекст
Ми розробляємо MVP CRM-системи для СТО. Команда складається з 2 розробників. Нам потрібно швидко розробляти функціонал, маючи чіткі межі між доменами (CRM, Billing, Warehouse).

## Рішення
Ми обираємо стиль **Модульний Моноліт**.
Всі модулі знаходяться в одному репозиторії та розгортаються як один процес, але логічно ізольовані на рівні коду.

## Наслідки
* Позитивні: Простий деплой, легкий рефакторинг, відсутність мережевих затримок між модулями.
* Негативні: Спільний стек технологій для всіх модулів.

## Альтернативи
* Мікросервіси: Відхилено через надмірну складність інфраструктури для поточної стадії проєкту.
